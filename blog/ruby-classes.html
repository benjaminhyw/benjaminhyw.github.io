Ruby Classes

Sunday, March 20th 2016

It's a bird! It's a plane! No, it's a Class!

Okay, lame intro aside... Yes! Today we are talking about Ruby Classes!

Classes are these cool things in ruby that allow you to store a lot of different information in instance variables, which then allow you to easily access that data in all of your different methods!

I know I know, I started with a Superman reference so you want a nerdy explanation... except, I don't know really know anything about Superman! Or DC, for that matter! I'm a Marvel fanboy.. *sigh* so, uh, yea, here's a very sloppy transition .. Oh hey, the X-Men! Alright!

As you may or may not know, the X-Men are a group of individuals with very cool super-abilities! So, let's make a class for the members and their abilities! No, this is not the mutant registration act...

When we create a class, we write all of the methods we want to be moving info inside of the class (begins line 2 ends line 20)

We start with the initialize method, where we create our instance variables.

Then we create all our other methods. In this case, I created 3 different methods. Each one essentially either greets a new member, greets a returning member, or says goodbye to a leaving member. Whenever a member leaves or joins, an automated message will be printed on the screen, including the members name and their ability. After you've done all of your class and methods, you can call the method on the class object!
